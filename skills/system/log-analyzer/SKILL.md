# 日志分析

请按以下步骤分析系统日志：

1. 调用 `get_system_logs` 获取最近 200 条日志
2. 筛选 ERROR、WARN 级别的日志条目
3. 按错误类型分组统计
4. 识别重复出现的错误模式
5. 分析时间分布（是否有集中爆发）

输出分析报告：

## 日志分析报告

### 错误统计
| 类型 | 数量 | 最近出现时间 |
|------|------|-------------|
| ... | ... | ... |

### 关键发现
- 列出主要错误模式
- 分析可能的根因

### 趋势
- 错误是否在增加？
- 是否有周期性模式？

### 建议
- 针对每个主要错误给出处理建议
