# 智控助手使用指南

智控助手是基于 AI 的管理工具，可以通过自然语言与系统交互，支持 MCP 工具调用和 Skill 扩展。

## 配置项详解

| 配置项 | 含义 | 说明 |
|--------|------|------|
| **后端代理** | 用于连接 AI 服务的代理 | 必须选择一个运行中的代理 |
| **供应商** | 指定使用哪个供应商的 AI | 默认为「跟随代理」，即使用代理配置的供应商 |
| **模型** | 指定使用的 AI 模型 | 默认为「跟随代理」，即使用代理配置的默认模型 |
| **会话** | 对话历史管理 | 可创建新会话或切换到历史会话 |

### 后端代理说明

| 要求 | 说明 |
|------|------|
| 必须运行中 | 代理必须在「运行中」状态才能使用 |
| 协议匹配 | 代理的协议需要与 AI 服务兼容（OpenAI 格式推荐） |

### 模型上下文配置

| 配置项 | 含义 | 影响 |
|--------|------|------|
| **上下文大小** | 模型能处理的最大 token 数 | 影响压缩触发阈值 |
| **最大工具调用轮次** | 单次对话中模型连续调用工具的最大次数 | 到达上限后模型自动总结回复 |
| **最大会话数** | 保存的历史会话数量上限 | 超出自动删除最早的会话，0 表示不限制 |

## 开始对话

1. 点击左侧菜单「**智控助手**」
2. 选择「**后端代理**」（必选）
3. （可选）指定供应商和模型
4. 在输入框输入问题
5. 点击「**发送**」或按 Enter

## 对话管理

| 操作 | 说明 |
|------|------|
| **新会话** | 点击「新会话」按钮清空当前对话 |
| **切换会话** | 点击会话下拉框选择历史会话 |
| **压缩上下文** | 当上下文接近上限时，点击「压缩」按钮整理对话 |

### 会话管理工具

AI 也可通过工具管理会话：

| 工具 | 功能 |
|------|------|
| `list_conversations` | 列出所有历史会话（含消息数和预览） |
| `delete_conversation` | 删除指定会话 |
| `clear_conversation` | 清空指定会话的消息历史 |

### 上下文管理说明

智控助手会跟踪对话上下文：
- 显示当前 token 使用量 / 最大值
- 当使用量超过 80% 时显示压缩提示
- 压缩后保留关键信息，删除冗余内容
- 可在「设置」中调整模型上下文大小

## 客户端斜杠命令

在智控助手输入框中输入 `/` 可触发自动补全，显示可用的**客户端命令**和**技能**。客户端命令直接在浏览器端处理，不发送到 AI。

| 命令 | 功能 | 说明 |
|------|------|------|
| `/help` | 显示帮助 | 列出所有可用命令和技能 |
| `/clear` | 清空对话 | 清除当前对话的显示内容 |
| `/new` | 新建会话 | 开启一个全新的对话会话 |
| `/compact` | 压缩上下文 | 将当前对话历史压缩为摘要，释放上下文空间 |
| `/model` | 切换模型 | 弹出模型选择器，快速切换当前使用的 AI 模型 |

**命令优先级**：客户端命令优先于技能。例如输入 `/help` 匹配客户端命令，不会发送到 AI；输入 `/health-check` 无客户端命令匹配时，正常触发对应技能。

## 中断 AI 响应

当 AI 正在生成回复时，可以随时中断：

| 方式 | 操作 |
|------|------|
| **键盘** | 按 **Escape** 键中断当前响应 |
| **按钮** | 点击发送按钮变为的「**停止**」按钮 |

中断后，AI 已生成的内容会保留，后续内容不再生成。

## 技能（Skill）系统

技能管理是智控助手的核心扩展机制，允许通过自然语言指令扩展 AI 的能力边界。

### 技能触发机制

技能有三种触发方式：

1. **用户手动触发**：输入 `/技能名` 或在技能面板点击
2. **AI 自动触发**：当用户请求匹配某个技能的 `trigger` 条件时，AI 会自动调用该技能
3. **AI 工具调用**：AI 在对话过程中通过 `invoke_skill` 工具主动加载技能

### 使用技能

| 方式 | 操作 |
|------|------|
| **斜杠触发** | 输入 `/技能名` 直接触发，如 `/health-check` |
| **自动补全** | 输入 `/` 后继续输入文字，自动匹配命令和技能，支持 ↑↓ 键盘导航 |
| **面板加载** | 点击输入框右侧「⚡」按钮，在技能面板中点击加载 |
| **AI 自动触发** | AI 根据技能的 `trigger` 条件自动判断是否需要调用 |

### 创建自定义技能

1. 点击「⚡」技能面板中的「+」按钮
2. 填写配置：

| 字段 | 含义 | 示例 |
|------|------|------|
| **名称** | 技能的唯一标识，英文 | `my-helper` |
| **描述** | 技能功能说明 | `帮助整理代码规范` |
| **触发条件** | AI 何时应自动使用此技能 | `用户要求检查代码规范、整理代码风格` |
| **内容** | 技能的详细指令（Markdown 格式） | 技能执行的具体步骤和规则 |

3. 点击「保存」

### 编辑和删除技能

- **编辑**：在技能面板中点击技能名称，修改后保存
- **删除**：在技能编辑页面点击「删除」按钮

### 触发条件配置

`trigger` 字段描述了技能应在什么场景下被使用，帮助 AI 判断是否需要调用：

```
trigger: 用户要求分析日志、查找错误、排查问题趋势
```

- 使用自然语言描述，多个条件用中文逗号分隔
- AI 会将用户请求与所有技能的触发条件进行匹配
- 匹配成功时，AI 必须先加载技能指令，再按指令执行

### 技能内容编写建议

| 建议 | 说明 |
|------|------|
| 明确步骤 | 使用编号列表描述执行步骤 |
| 指定工具 | 如需调用 API，列出需要使用的工具名 |
| 输出格式 | 说明期望的输出格式（表格、列表等） |
| 边界条件 | 描述异常情况的处理方式 |

### MCP 工具与技能配合

如果技能需要调用 MCP 工具，在技能内容中说明即可：
```markdown
1. 调用 `web_search` 搜索相关信息
2. 分析搜索结果
3. 汇总输出
```

AI 会自动在技能执行过程中调用对应的 MCP 工具。

MCP 服务器可为 AI 提供额外工具能力。系统预置了 10+ 个常用 MCP 配置（Filesystem、Context7、Chrome DevTools 等），详见 `/guide-mcp` 技能。

## 子智能体委派

智控助手支持将复杂任务拆分为多个子任务，并行委派给子智能体执行。

### 使用方式

直接用自然语言描述需要完成的任务，AI 会根据需要自动调用 `delegate_task` 工具：

- "帮我分析一下最近的请求日志，同时检查密钥健康状态" → 自动拆分为两个并行子任务
- "帮我创建一个新供应商，然后测试连通性" → 自动拆分为串行子任务

### 子代理角色

子代理支持权限分化，不同权限决定可用的工具和操作范围：

| 权限 | 适用场景 | 能力 |
|------|----------|------|
| `full` | 通用任务（默认） | 继承全局配置 |
| `readonly` | 搜索、分析、审查、规划 | 只读，无法修改文件或执行命令 |
| `writer` | 编码、重构、跑测试 | 可写项目文件，可执行命令（受策略控制） |

权限可通过 `delegate_task` 的 `role` 参数（统一权限）或 `goal_roles` 数组（每个子任务不同权限）指定。详细的委派指南可通过 `/delegate` 技能查看。

### 子任务管理

子任务创建后，可通过以下工具查看和管理：

| 工具 | 功能 | 说明 |
|------|------|------|
| `list_tasks` | 列出子任务 | 可按状态筛选（created/running/completed/failed/stopped） |
| `get_task` | 查看详情 | 获取单个子任务的完整信息 |
| `stop_task` | 终止任务 | 停止正在运行的子任务 |
| `message_task` | 追加消息 | 向已完成或失败的子任务追加消息，可指定额外轮次继续执行 |
| `list_agents` | 列出代理身份 | 查看所有可用的代理身份（用于 `delegate_task` 的 `agent` 参数） |

### 子任务安全限制

子智能体运行在受限沙盒中：递归委派被阻止，写入隔离在沙盒目录内，危险命令由执行策略引擎拦截。详见 `/guide-settings` 第三节（子智能体设置）和第六节（执行策略）。

### 子任务状态

子任务卡片会在对话中实时显示状态变化：

| 状态 | 含义 |
|------|------|
| **created** | 已创建，等待执行 |
| **running** | 正在执行 |
| **completed** | 执行完成，显示结果摘要 |
| **failed** | 执行失败，显示错误信息 |
| **stopped** | 被手动终止 |

## Agent 身份管理

Agent 身份用于定义子代理的人设和行为风格，可通过 `delegate_task` 的 `agent` 参数指定。

| 工具 | 功能 |
|------|------|
| `list_agents` | 列出所有可用的代理身份 |
| `get_agent` | 获取单个 Agent 的详细信息 |
| `create_agent` | 创建新的 Agent 身份（名称、人设、颜色、角色） |
| `update_agent` | 修改已有 Agent 身份的配置 |
| `delete_agent` | 删除非系统级 Agent 身份 |
| `reload_agents` | 从磁盘重新加载所有 Agent 定义 |

## Agent 人设（SOUL.md）

智能体支持自定义人设（Persona），通过 SOUL.md 定义智能体的角色、性格和行为准则。

| 操作 | 方式 |
|------|------|
| **查看** | 在「设置」→「记忆」中查看当前人设，或使用 `get_memory` 工具 |
| **修改** | 在「设置」→「记忆」中编辑，或使用 `update_soul` 工具 |
| **生效** | 人设内容会在每次对话时注入系统提示词，修改后立即生效 |

## 工具审批

当 AI 调用高权限工具（如写入文件、执行命令）时，系统会弹出审批请求：

| 操作 | 说明 |
|------|------|
| **批准** | 允许本次工具调用执行 |
| **拒绝** | 阻止本次工具调用，AI 会收到拒绝提示 |

工具审批权限可在「设置」中配置，也可通过工具权限分级自动控制。

## 其他能力

- **客户端配置管理**：检测、配置、备份恢复 Claude Code / Codex 的代理连接，详见 `/guide-client-config`
- **MCP 预设推荐**：通过 `get_mcp_presets` 获取预置 MCP 服务器列表并引导安装，详见 `/guide-mcp`

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| **Enter** | 发送消息 |
| **Shift + Enter** | 输入换行 |
| **Escape** | 中断 AI 响应 / 关闭自动补全 |
| **/** | 触发命令和技能自动补全 |
| **↑↓** | 在自动补全列表中导航选择 |
| **Enter**（补全列表中） | 确认选中项 |

## 使用示例

智控助手支持自然语言交互，以下是典型场景：

| 指令示例 | 说明 |
|----------|------|
| "帮我看看系统状态" | 获取代理、密钥、供应商运行状态 |
| "启动所有代理" / "停止代理 xxx" | 管理代理启停 |
| "帮我新建一个 OpenAI 供应商" | 创建供应商 |
| "分析最近的请求日志，看看有什么问题" | 排查请求问题 |
| "帮我开启开机自启动" | 管理开机自启 |
| "添加一条允许 npm install 的执行策略" | 管理命令执行策略 |
| "帮我把代理配置写入 Claude Code" | 客户端配置管理 |
| "测试一下代理 xxx 的连通性" | 客户端连接测试 |
| "创建一个代码审查 Agent" | Agent 身份管理 |
| "推荐几个好用的 MCP 服务器" | MCP 预设推荐 |
