# 代理诊断

当用户报告代理连接问题时，请按以下步骤诊断：

1. 调用 `get_proxies` 获取所有代理状态
2. 对有问题的代理，调用 `get_recent_requests` 查看最近请求
3. 调用 `get_system_logs` 搜索相关错误日志
4. 分析请求延迟和成功率

诊断清单：

### 连接检查
- 代理是否在运行？
- 端口是否正确监听？
- 认证配置是否正确？

### 转发检查
- 目标供应商是否可达？
- API Key 是否有效？
- 协议转换是否正常？

### 性能检查
- 平均延迟是否正常？
- 是否有超时请求？
- 是否有密钥限流？

输出诊断结果和修复建议。
