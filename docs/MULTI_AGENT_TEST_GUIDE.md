# 多智能体协作功能测试指南

## 测试前准备

1. 启动服务：`node server.js`
2. 确保至少一个代理处于「运行中」状态
3. 打开智控助手页面，发送一条消息激活代理上下文
4. 打开浏览器 DevTools Console 观察日志

---

## 一、基础委派功能

### 1.1 单任务委派

**操作**：在智控助手中输入：
```
帮我查看当前系统的代理状态和供应商列表
```

**预期**：
- AI 自动调用 `delegate_task` 工具
- 对话中出现子任务卡片，显示 1 个任务
- 状态从 created → running → completed
- 任务卡片显示绿色完成状态和结果摘要

### 1.2 多任务并行委派

**操作**：
```
同时帮我做三件事：1) 查看系统日志最近的错误 2) 检查密钥健康状态 3) 查看当前用量统计
```

**预期**：
- 出现包含 3 个子任务的委派卡片
- 3 个任务并行执行（状态同时为 running）
- 依次完成，每个显示各自的摘要
- 最终显示 "已完成 3/3"

### 1.3 混合角色委派

**操作**：
```
帮我做两件事：1) 搜索项目中所有使用了 async/await 的文件 2) 在沙盒中创建一个测试报告文件
```

**预期**：
- AI 应选择 explore 角色处理搜索任务
- AI 应选择 implementer 角色处理文件创建任务
- 搜索任务只使用只读工具（read_file, grep_search 等）
- 创建任务可以使用 write_file（在沙盒内）

---

## 二、角色系统

### 2.1 explore 角色

**操作**：
```
用 explore 角色分析 server.js 的代码结构，列出所有 API 端点
```

**预期**：
- 子代理只能调用只读工具
- 如果尝试 write_file/edit_file，返回 auto-deny 错误
- 如果尝试 execute_command，返回 auto-deny 错误
- 成功完成分析任务

### 2.2 implementer 角色

**操作**：
```
用 implementer 角色在沙盒中创建一个 hello.txt 文件，内容为 "Hello World"
```

**预期**：
- 子代理可以调用 write_file
- 文件创建在沙盒目录内（不在项目目录中）
- 任务完成后沙盒延迟清理（30 分钟内可通过 message_task 续接）

### 2.3 reviewer 角色

**操作**：
```
用 reviewer 角色审查 lib/multi-agent/delegate.js 的错误处理是否完整
```

**预期**：
- 子代理只使用只读工具
- 输出按严重程度分类（高/中/低）
- 每条问题给出文件位置和修复建议

### 2.4 goal_roles 差异化

**操作**：
```
帮我做三件事，分别用不同角色：
1) (explore) 搜索项目中所有的 TODO 注释
2) (reviewer) 检查 server.js 的输入验证
3) (implementer) 在沙盒中创建一个分析报告
```

**预期**：
- 3 个子任务分别使用指定的角色
- 每个子任务的工具权限不同
- 任务卡片上显示角色标签

## 三、执行策略

### 3.1 Allow 命令（直接执行）

**操作**：
```
执行命令：git status
```

**预期**：
- 命令直接执行，不弹出审批
- 返回 git 状态信息
- 日志中无 `[exec-policy]` 相关记录

### 3.2 Prompt 命令（需确认）

**操作**：
```
执行命令：npm install express
```

**预期**：
- 弹出审批对话框，显示三个按钮：
  - ✅ 本次批准
  - 🔒 本次会话批准
  - ❌ 拒绝
- 显示匹配规则信息（如 "npm install"）
- 点击「本次批准」→ 命令执行，返回结果
- 再次执行 `npm install lodash` → 再次弹出审批（未记忆）

### 3.3 会话级批准

**操作**：
```
执行命令：npm install express
```
点击「🔒 本次会话批准」

然后：
```
执行命令：npm install lodash
```

**预期**：
- 第一次弹出审批，点击会话批准
- 第二次 `npm install` 前缀匹配，直接执行，不弹审批
- 日志显示 `[exec-policy] 会话批准: npm install`

### 3.4 Forbidden 命令（直接拒绝）

**操作**：
```
执行命令：sudo apt update
```

**预期**：
- 不弹审批，直接返回错误
- 错误信息包含 `[FORBIDDEN]` 和原因说明
- 日志记录拒绝事件

### 3.5 危险删除命令

**操作**：
```
执行命令：rm -rf /
```

**预期**：
- 直接拒绝，不弹审批
- 返回 `[FORBIDDEN] 命令被安全策略禁止: rm -rf /（删除根目录）`

### 3.6 未知命令

**操作**：
```
执行命令：mycustomtool --version
```

**预期**：
- 弹出审批（未知命令默认 prompt）
- 显示 "未知命令，需用户确认"
- 可以批准或拒绝

### 3.7 策略测试工具

**操作**：在 AI 对话中：
```
帮我测试以下命令的执行策略：git status, git push --force, rm -rf /, npm install, curl http://example.com
```

**预期**：
- AI 调用 `test_exec_policy` 工具
- 返回每个命令的决策结果：
  - `git status` → allow
  - `git push --force` → forbidden
  - `rm -rf /` → forbidden
  - `npm install` → prompt
  - `curl http://example.com` → prompt

### 3.8 子代理执行策略

**操作**：
```
委派一个子任务：执行命令 node --version 查看 Node 版本
```

**预期**：
- 子代理执行 `node --version`
- 策略引擎返回 prompt（node 命令）
- 子代理自动批准并记录日志
- 命令执行成功，返回版本号
- 系统日志中出现 `[exec-policy] 子代理自动批准需确认命令: node --version`

## 四、进度回报

### 4.1 实时进度显示

**操作**：
```
帮我分析 server.js 的代码质量，同时检查 lib/ 目录的模块依赖
```

**预期**：
- 子任务卡片在 running 状态时显示进度行
- 进度格式：`轮次 N · 工具名`
- 每次工具调用后进度实时更新
- 任务完成后进度行消失

### 4.2 任务管理页面进度

**操作**：在上述任务执行期间，切换到「任务管理」页面

**预期**：
- 运行中的任务显示脉冲动画圆点
- 任务详情中显示当前轮次和工具信息
- 统计卡片中「运行中」数量正确

---

## 五、失败重试

### 5.1 网络错误重试

**操作**：停止后端代理，然后在智控助手中委派一个任务

**预期**：
- 子代理 LLM 调用失败
- 等待 2 秒后自动重试第 1 次
- 再等待 4 秒后自动重试第 2 次（如果 maxRetries >= 2）
- 重试次数耗尽后标记为 failed
- 任务卡片显示失败状态和错误信息

### 5.2 重试后恢复

**操作**：在子代理重试期间，重新启动后端代理

**预期**：
- 重试的 LLM 调用成功
- 子代理继续正常执行
- 最终标记为 completed

---

## 六、层级取消

### 6.1 停止单个任务

**操作**：委派多个子任务，在其中一个运行中时，点击任务卡片上的「停止」按钮

**预期**：
- 被停止的任务状态变为 stopped
- 其他任务不受影响，继续运行
- 任务卡片显示「已停止」状态

### 6.2 批量取消（关闭对话）

**操作**：委派多个子任务，在任务运行中时，刷新浏览器页面或关闭标签页

**预期**：
- 服务端检测到客户端断开
- 日志显示 `[assistant] 客户端断开，停止 N 个子任务`
- 所有关联子任务状态变为 stopped
- 子代理的 LLM 请求被中断

---

## 七、message_task 续接

### 7.1 续接已完成的任务

**操作**：
```
委派一个任务：分析 server.js 的路由结构
```
等待任务完成后：
```
继续刚才的任务，补充分析中间件的执行顺序
```

**预期**：
- AI 调用 `message_task` 工具，传入原任务 ID 和新消息
- 子代理从上次对话上下文继续执行
- 保留之前的分析结果，在此基础上补充
- 最终返回更完整的分析

### 7.2 续接失败的任务

**操作**：委派一个任务，等它失败后：
```
重试刚才失败的任务，换个方式完成
```

**预期**：
- AI 调用 `message_task` 工具
- 子代理从失败点继续，尝试新策略
- 如果成功，任务状态更新为 completed

### 7.3 续接过期

**操作**：委派一个任务，等待 30 分钟以上，然后尝试续接

**预期**：
- `message_task` 返回错误：「对话历史已过期，无法续接」
- 沙盒目录已被清理

---

## 八、任务管理页面

### 8.1 页面导航

**操作**：点击左侧菜单「任务管理」

**预期**：
- 显示统计概览卡片（运行中/已完成/失败/平均耗时）
- 显示任务列表
- 导航徽标显示运行中任务数

### 8.2 搜索过滤

**操作**：在搜索框输入关键词，切换状态筛选

**预期**：
- 搜索框实时过滤（按目标和摘要文本匹配）
- 状态下拉筛选对应状态的任务
- 无匹配时显示「无匹配任务」

### 8.3 任务详情展开

**操作**：点击任意任务卡片

**预期**：
- 展开显示详情区域
- 摘要以 Markdown 格式渲染（支持标题、表格、代码块等）
- 错误信息以红色左边框样式显示
- 显示 ID、角色、创建时间、耗时
- 再次点击折叠

### 8.4 实时更新

**操作**：在任务管理页面，从智控助手委派一个新任务

**预期**：
- 新任务自动出现在任务列表顶部
- 运行中任务的状态实时更新
- 统计卡片数字实时变化
- 导航徽标数字变化

### 8.5 清理已完成任务

**操作**：点击「清理已完成」按钮

**预期**：
- 弹出确认对话框，显示将清理的任务数量
- 确认后删除所有 completed/failed/stopped 的任务
- 列表刷新，统计更新

### 8.6 停止运行中任务

**操作**：在任务管理页面，点击运行中任务的「停止」按钮

**预期**：
- 任务状态变为 stopped
- 子代理的 LLM 请求被中断
- 列表实时更新

## 十、配置与设置

### 10.1 子智能体设置

**操作**：进入「设置」→「子智能体」，修改以下配置：
- 最大并发数改为 5
- 最大轮次改为 10
- 超时时间改为 600
- 失败重试次数改为 2

**预期**：
- 修改后立即生效（无需保存按钮）
- 显示保存成功的 toast 提示
- 下一次委派任务使用新配置

### 10.2 执行策略设置

**操作**：进入「设置」→「执行策略」

**预期**：
- 显示默认规则统计（allow/prompt/forbidden 数量）
- 显示自定义规则统计
- 测试输入框可输入命令并显示决策结果
- 自定义规则列表可删除

### 10.3 策略测试

**操作**：在执行策略设置的测试框中输入：
```
git push --force origin main
```
点击「测试」

**预期**：
- 显示 `Forbidden — 禁止`
- 显示匹配规则：`git push --force`
- 显示描述：`Git 强制推送`
- 红色背景高亮

## 十一、边界场景

### 11.1 空目标委派

**操作**：
```
帮我委派任务
```
（不指定具体目标）

**预期**：
- AI 应主动询问具体要做什么
- 不会调用 `delegate_task` 传入空目标

### 11.2 超过 10 个子任务

**操作**：尝试一次委派 11 个子任务

**预期**：
- 返回错误：「一次最多委派 10 个子任务」

### 11.3 子代理递归委派

**操作**：委派一个子任务，子任务中 AI 尝试调用 `delegate_task`

**预期**：
- 子代理看不到 `delegate_task` 工具（被 blocked）
- 如果 somehow 调用，返回错误

### 11.4 沙盒外写入

**操作**：委派一个 implementer 子任务，目标为：
```
在 C:/ 目录下创建一个 test.txt 文件
```

**预期**：
- write_file 被沙盒拦截
- 返回 `[SCOPE] 写入被限制在沙盒目录内` 错误
- 子代理换其他方式或报告失败

### 11.5 上下文溢出

**操作**：委派一个需要大量工具调用的子任务，设置最大轮次为 50

**预期**：
- 当消息总长度超过 150,000 字符时自动裁剪
- 保留最近 4 轮完整对话
- 任务继续执行不中断

---

## 十二、WebSocket 实时性

### 12.1 多标签页同步

**操作**：打开两个浏览器标签页，都在智控助手页面

**预期**：
- 在标签页 A 委派任务
- 标签页 B 的任务管理页面也实时更新
- 两个标签页的导航徽标同步

### 12.2 断线重连

**操作**：在任务运行中，断开网络 5 秒后恢复

**预期**：
- WebSocket 自动重连
- 重连后任务状态自动同步
- 不会丢失已完成的任务信息
